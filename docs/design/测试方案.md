# 前端代码测试方案

> 项目：《昭君：一路向北》
> 版本：V1.0 | 2026年7月

---

## 一、测试目标

1. **功能测试**：验证核心模块（GameEngine、AudioManager、SaveManager、Store）的功能正确性
2. **集成测试**：验证 Phaser 场景与 React HUD 的通信机制
3. **UI 测试**：验证 HUD 组件的渲染和交互
4. **性能测试**：验证场景切换和资源加载的性能

---

## 二、测试环境

- **测试框架**：Vitest（Vite 生态，快）
- **断言库**：expect（内置）
- **模拟工具**：vi.fn()、vi.mock()
- **覆盖率**：目标 80%+

---

## 三、测试范围

### 3.1 单元测试（Unit Tests）

| 模块 | 测试内容 | 优先级 |
|------|----------|--------|
| **GameEngine** | 初始化、场景切换、销毁 | P0 |
| **AudioManager** | 音频加载、播放、停止、音量控制 | P0 |
| **SaveManager** | 存档、读档、清除、导入导出 | P0 |
| **gameStore** | Zustand store 的状态更新、持久化 | P1 |
| **collectiblesStore** | 收集品解锁、进度计算 | P1 |
| **BaseScene** | 对话显示、过渡转场 | P1 |

### 3.2 集成测试（Integration Tests）

| 场景 | 测试内容 | 优先级 |
|------|----------|--------|
| **Phaser + React 通信** | 事件触发、状态同步 | P0 |
| **场景切换流程** | 从序章到第一章的完整流程 | P0 |
| **收集品流程** | 点击收集品 → 解锁 → 更新进度 | P1 |
| **存档流程** | 游戏结束 → 存档 → 读档恢复 | P1 |

### 3.3 端到端测试（E2E Tests）

| 流程 | 测试内容 | 优先级 |
|------|----------|--------|
| **完整游戏流程** | 从开始到第一章结束 | P1 |
| **HUD 交互** | 点击按钮、打开旅行册、设置 | P2 |

## 五、测试执行计划

1. **安装依赖**：`npm install -D vitest @testing-library/react @testing-library/jest-dom jsdom`
2. **配置 Vitest**：`vitest.config.ts`
3. **编写测试用例**：按模块逐一编写
4. **运行测试**：`npm test`
5. **查看覆盖率**：`npm test -- --coverage`
6. **修复问题**：根据测试结果调整代码
7. **代码审查**：测试通过后进行代码审查
