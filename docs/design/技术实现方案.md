### 3.2 场景基类（BaseScene）

**职责**：
- 提供统一的章节结构
- 管理背景图、角色、对话
- 处理互动点击
- 与 React UI 组件通信（通过自定义事件）

```typescript
// scenes/BaseScene.ts
export abstract class BaseScene extends Phaser.Scene {
  protected chapterId: string;
  protected background: Phaser.GameObjects.Image;
  protected character: Phaser.GameObjects.Sprite;
  protected dialogueBox: Phaser.GameObjects.Text;

  constructor(chapterId: string) {
    super({ key: chapterId });
    this.chapterId = chapterId;
  }

  preload() {
    // 加载本章背景图、角色图、音频
    this.load.image('bg', `assets/images/chapters/${this.chapterId}/bg.png`);
  }

  create() {
    // 创建背景
    this.background = this.add.image(0, 0, 'bg').setOrigin(0);

    // 创建角色（如果需要）
    // this.character = this.add.sprite(...)

    // 初始化交互区域
    this.setupInteractions();

    // 发送事件通知 React UI 显示对白
    this.events.emit('dialogue:start', { text: '...' });
  }

  protected abstract setupInteractions(): void;
  protected abstract onCollectibleClicked(collectibleId: string): void;

  // 公共方法：显示对白
  showDialogue(text: string, duration?: number) {
    this.dialogueBox = this.add.text(100, 600, text, {
      fontSize: '24px',
      color: '#ffffff',
      backgroundColor: '#00000080',
      padding: { x: 20, y: 10 }
    });
  }

  // 公共方法：隐藏对白
  hideDialogue() {
    if (this.dialogueBox) this.dialogueBox.destroy();
  }
}
```

### 3.4 状态管理（Zustand）

**职责**：
- 管理当前章节、进度
- 收集品解锁状态
- 游戏设置
- 存档/读档

```typescript
// store/gameStore.ts
import { create } from 'zustand';

interface GameState {
  currentChapter: string;
  progress: Record<string, boolean>; // 章节完成状态
  settings: {
    volume: number;
    subtitles: boolean;
  };
  actions: {
    setChapter: (chapterId: string) => void;
    completeChapter: (chapterId: string) => void;
    updateSettings: (settings: Partial<GameState['settings']>) => void;
  };
}

export const useGameStore = create<GameState>((set, get) => ({
  currentChapter: 'prologue',
  progress: {},
  settings: {
    volume: 0.8,
    subtitles: true,
  },
  actions: {
    setChapter: (chapterId) => set({ currentChapter: chapterId }),
    completeChapter: (chapterId) => set((state) => ({
      progress: { ...state.progress, [chapterId]: true }
    })),
    updateSettings: (newSettings) => set((state) => ({
      settings: { ...state.settings, ...newSettings }
    })),
  },
}));
```

```

   ↓ 自定义事件（CustomEvent）
React 组件（HUD、对话框）
   ↓ state updates

```

**实现方式**：
- Phaser 通过 `this.events.emit('dialogue:start', data)` 发送事件
- React 使用 `window.addEventListener` 监听，或者通过 Zustand 共享状态

## 六、Phaser + React 集成方案

### 方案 A：Phaser 作为 React 组件的子元素

```tsx
// App.tsx
import { useEffect, useRef } from 'react';
import { GameEngine } from './core/GameEngine';

export default function App() {
  const gameRef = useRef<GameEngine | null>(null);
  const containerRef = useRef<HTMLDivElement>(null);

  useEffect(() => {
    if (containerRef.current) {
      gameRef.current = new GameEngine();
      gameRef.current.init(containerRef.current);
    }
    return () => gameRef.current?.destroy();
  }, []);

  return (
    <div className="w-full h-screen bg-black">
      <div id="game-container" ref={containerRef} className="w-full h-full" />
      <HUD />
    </div>
  );
}
```

**优点**：React 控制布局，HUD 可以轻松覆盖在 Phaser 画布上
**缺点**：Phaser 事件和 React 事件需要桥接

---

### 方案 B：Phaser 作为全屏容器，React UI 通过 z-index 覆盖

```tsx
// App.tsx
export default function App() {
  return (
    <div className="relative w-full h-screen overflow-hidden">
      <PhaserGame /> {/* 全屏，z-index: 1 */}
      <HUD className="absolute inset-0 z-10 pointer-events-none" />
    </div>
  );
}
```

**优点**：Phaser 完全控制游戏画面，React 只负责 UI
**缺点**：事件转发稍复杂

### 7.2 收集品系统

```typescript
// store/collectiblesStore.ts
interface Collectible {
  id: string;
  name: string;
  description: string;
  chapterId: string;
  unlocked: boolean;
}

interface CollectiblesState {
  items: Record<string, Collectible>;
  unlocked: string[];
  actions: {
    unlock: (id: string) => void;
    isUnlocked: (id: string) => boolean;
    getProgress: () => number;
  };
}

export const useCollectiblesStore = create<CollectiblesState>((set, get) => ({
  items: {
    bronze_mirror: { id: 'bronze_mirror', name: '铜镜碎片', description: '...', chapterId: 'chapter01', unlocked: false },
    wei_stone: { id: 'wei_stone', name: '圆润的石头', description: '...', chapterId: 'chapter02', unlocked: false },
    // ... 其他收集品
  },
  unlocked: [],
  actions: {
    unlock: (id) => {
      const item = get().items[id];
      if (item && !get().unlocked.includes(id)) {
        set((state) => ({
          items: { ...state.items, [id]: { ...item, unlocked: true } },
          unlocked: [...state.unlocked, id],
        }));
        // 触发收集动画
        window.dispatchEvent(new CustomEvent('collectible:unlocked', { detail: item }));
      }
    },
    isUnlocked: (id) => get().unlocked.includes(id),
    getProgress: () => Math.round((get().unlocked.length / Object.keys(get().items).length) * 100),
  },
}));
```

// utils/transitions.ts
import gsap from 'gsap';

export function chapterTransition(onComplete: () => void) {
  const overlay = document.getElementById('transition-overlay');
  if (!overlay) return;

  gsap.timeline()
    .to(overlay, { opacity: 1, duration: 0.8, ease: 'power2.inOut' })
    .call(onComplete)
    .to(overlay, { opacity: 0, duration: 0.8, ease: 'power2.inOut' });
}

## 九、性能优化

1. **图片懒加载**：使用 Phaser 的 `this.load.image()` 延迟加载
2. **音频预加载**：在章节开始前预加载环境音
3. **场景缓存**：已完成的场景可以保留在内存中（需要更多内存）
4. **按需渲染**：Phaser 默认行为，无需特殊配置
