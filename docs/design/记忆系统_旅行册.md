# 记忆系统：旅行册设计

> 《昭君：一路向北》创新亮点
> 版本：V1.0 | 2026年7月

---

## 核心概念

玩家不会获得金币或装备，而是收集"记忆碎片"——一阵秦岭的雨声、一句离别的话语、一片草原的落叶、一首胡笳曲。

抵达单于庭后，这些碎片会自动组合成一本专属于本次旅程的《一路向北》电子画册。由于玩家停留的地点、触发的事件和收集内容不同，**每一次游玩形成的画册内容都会略有差异**。

## 收集逻辑

```typescript
interface CollectibleRule {
  id: string;
  chapterId: string;
  trigger: 'hotspot' | 'auto' | 'interaction';
  condition?: string;   // 可选的触发条件（如"停留超过10秒"）
}
```

- **hotspot**：点击场景中的热区触发
- **auto**：章节完成时自动获得（如终章）
- **interaction**：完成某互动后自动获得（如过河小游戏后）

```
┌─────────────────────────────────┐
│ 一路向北                        │
│ 旅行册                          │
├─────────────────────────────────┤
│ [铜镜] [渭河之水] [秦岭夜雨]      │

│ [归途的沉默]                     │
└─────────────────────────────────┘
```

- 已解锁：彩色图标 + 描述文字
- 未解锁：灰色剪影 + "？？？"
- 点击已解锁 → 弹窗显示详情（名称 + 描述 + 所属章节）

## 记忆合成（第九章触发）

当玩家完成第九章抵达王庭时：

1. 所有已收集的碎片从背包中飞出
2. 在屏幕中央汇聚、旋转、重组
3. 渐变成《一路向北旅行册》封面
4. 玩家可以点击查看所有收集品
5. 同时触发终章

**动态性**：若某些收集品未获得，旅行册对应格子留空，但合成动画照常播放——"你的旅程，缺了点什么"。

## 数据持久化

```typescript
interface SaveData {
  version: string;
  inventory: {
    collected: string[];   // 收集品ID列表
    chapterProgress: {
      [chapterId: string]: {
        completed: boolean;
        collectiblesFound: string[];
        timestamp?: number;
      };
    };
  };
  settings: {
    ambientVolume: number;
    musicVolume: number;
    voiceVolume: number;
    textSpeed: 'slow' | 'medium' | 'fast';
  };
}
```

- 自动保存到 `localStorage`（或 IndexedDB）
- 跨设备同步（后续版本可选）

## 设计思考

这个系统让玩家真正"带走"一份属于自己的昭君之旅。
每一次游玩因为选择停留的地方不同，收集品不同，画册也不同。
这既增强了重玩价值，也让玩家真正理解——
**昭君的旅程不是一条固定的路，而是她一次一次的选择。**
